# Aphelion Promise Ledger

This ledger tracks public promises that can be mistaken for current behavior.
Status values:

- `implemented`: covered by live code and tests

Every broad README or requirements claim should map here when the implementation
surface is not obvious from the code.

| Promise | Status | Current behavior | Next implementation target |
|---|---:|---|---|
| Single Linux daemon for Telegram, sessions, tools, and delivery | implemented | Telegram polling, SQLite sessions, progress, slash commands, maintenance loops, and delivery guards exist. | Keep covered by runtime and deployment tests. |
| Provider support for Anthropic, OpenAI, OpenRouter, Gemini, and Ollama | implemented | Native adapters, streaming, tool-call mapping, model-slot routing, and failover-chain config exist for these providers. | Keep provider request/stream/failover tests current; add live readiness probes if operator experience needs them. |
| Provider prompt caching for Anthropic and OpenAI | implemented | Anthropic receives explicit structured `cache_control: ephemeral` on selected system/tool blocks, with config-backed strategy/off and TTL validation. Official OpenAI API requests rely on OpenAI's automatic exact-prefix prompt caching, so Aphelion uses the same cache-aware stable-prefix/dynamic-tail prompt layout without emitting an OpenAI transport cache-control field. OpenAI-compatible custom endpoints stay conservative unless they become a first-class cache contract. | Keep cache-shaping tests current as prompt sections evolve. |
| Single static binary | implemented | Normal build remains a fast developer binary; `make build-static` and release CI produce a statically linked Linux binary. | Keep static proof covered by release CI. |
| String anonymization / provider-visible identity control | implemented | `identity.user_agent`, `identity.face_name`, and `identity.governor_name` are configurable; anonymous profile suppresses project-specific default user agents unless the operator explicitly sets one. Operational copy and provider-visible instructions avoid avoidable project/face labels outside intentional self-awareness and face-identity surfaces. | Keep leak tests current and separate local relationship names from outbound protocol identity. |
| Config schema distinguishes live keys from future knobs | implemented | `config.example.toml` is the live operator schema and is load-tested without ignored-key warnings. Broader requirements entries are explicitly future design notes; ignored keys remain valid but are surfaced as warnings in config load, startup/check-config logs, and `/health diagnose`. | Keep new live keys in `config.Config`, `config.example.toml`, config tests, and operator projections together. |
| GitHub App credential checks | implemented | Configured GitHub App PEM files are explicit host secrets, hidden from non-admin/durable sandbox profiles, and usable through `aphelion github-app status` and `aphelion github-app token` for redacted verification or intentional manual token output. Tokens are minted through GitHub's app JWT and installation-token flow and are not persisted. | Keep this as a narrow credential resolver unless a governed GitHub operation boundary is added. |
| Sandbox isolation for non-admin/durable execution | implemented | Config-backed bubblewrap profiles isolate roots, hide secrets, clear env, drop caps, enforce network denial, and can enforce isolated `network=allowlist` through the helper-backed Linux netns+nftables backend when host prerequisites are present. If the backend is unavailable, execution refuses instead of using host networking. Hostname allowlist entries compile to IPv4/port rules at run start; this is not HTTP Host/SNI policy. | Keep `aphelion sandbox-net check`, readiness projections, and runner tests aligned with backend behavior. |
| Native constrained file tools and web fetch | implemented | Scoped `read_file`, `write_file`, `list_dir`, `search`, and `fetch_url` exist; Linux file operations open approved roots and targets through no-follow descriptor-relative traversal beneath sandbox or `file_access` roots, file paths are checked against hidden paths, fetch honors profile network denial, and isolated `allowlist` fetches are app-enforced against resolved IP/port destinations, including redirects and non-admin host-private/special destination rejection. | Keep native fetch and process sandbox network semantics documented together without implying trusted host network isolation, HTTP Host/SNI identity enforcement, hard-link provenance, or non-Linux filesystem parity. |
| Telegram media handling without processing-choice buttons | implemented | Ordinary Telegram media routes immediately with session/reference retention by default. Voice/audio/video are normalized to agent-decide, ordinary photos/text/PDF-style documents avoid blocking retention selectors, and non-blocking keep buttons remain available for permanent/local retention. Exceptional files such as archives, executables, secrets, or oversized uploads still ask first. | Add provider-specific live probes for direct audio/video analysis quality if Gemini/native media use expands. |
| External tools: process/subprocess only | implemented | External manifest loading, install/audit/probe/verify records, registration, grant-gated listing, sandboxed `process`/`subprocess` execution, stale detection, rollback, and uninstall are implemented. `container` and `workspace_runner` manifests are importable/diagnosable only and are not callable. | Keep unsupported modes precise and non-callable unless a concrete governed outpost workflow justifies a full runtime. |
| Tailnet declarations and grant-binding projection | implemented | Durable-child tailnet declarations, local surface registry rows, grant-binding evidence, drift/rollback projection, Telegram `/tailnet grants`, `/status`, `/health diagnose`, Tailnet-private JSON mirrors, and `durable-agent provision` exist. The provisioner installs the Aphelion binary and remote bootstrap over Tailscale SSH, starts a child user service, and lets the child poll the parent Tailnet `/control` plane for enrollment, policy, parent conversation, review artifacts, and acknowledgements. Parent control-plane authority is bound to the Tailnet stable node identity observed by tsnet when accepting enrollment or the first valid accepted control request for an active enrollment with no stored node identity, and peer-identified control requests must continue satisfying declared child hostname/tag policy. | Keep Tailnet private and evidence-backed; add live Tailscale policy mutation only for a governed workflow that needs it. |
| Remote host control over Tailnet | implemented | Durable agents can receive a grant-bound `remote_host` tool for OpenSSH-over-Tailnet access to a macOS or Linux host. The grant uses `local_device` authority, `tailnet_host:<host>` targets, typed `remote_host` contract bounds, optional `tool_invocation` selectors, and records allowed, blocked, failed, and completed invocations as grant evidence. `codex_exec` runs Codex over SSH on the remote host using that host's own Codex installation and credentials. This does not install Aphelion on macOS or make the host a remote child. | Keep the lane explicit and grant-bound; add Telegram shortcuts only if they preserve the same typed grant flow. |
| Mission review without autonomous continuation | implemented | `/mission` and mission candidate/review surfaces can preserve objectives and surface review material. Mission state does not grant self-continuation, new capabilities, or external authority. | Keep self-continuation and broad Mission Control research-only unless separately justified by a governed workflow. |
| Authority/status/diagnosis consistency | implemented | Typed proposals, approval leases, grants, capability authority, auto-approval leases, TES, `/status`, `/health trace`, `/health diagnose`, and CLI `authority doctor`/`authority repair` project the same authority finding IDs. Findings separate human `suggested_repair` from typed `apply_action`/`apply_scope`; `authority repair --apply --finding <id>` applies only exact, local, fail-closed `applicable=true` repairs for expired leases/grants and invalid Tailnet grant bindings. Broader re-offer, escalation, and review work remains preview-only. | Keep finding IDs, projection checks, and repair tests current as new authority records are added. |
| Release install/update path | implemented | Release workflow builds the static binary, install/update scripts run park-restart, config preflight, init, and verify-deploy. | Keep release scripts covered by deploy checks. |
